About SWordy

What is SWordy?

SWordy is an Office add-in which allows data import from Stata into a Microsoft Word document. Data import is easy and fast because SWordy remotely connects to Stata for retrieving data, without producing any intermediate file. With SWordy you can also create "automatic documents", namely Word documents that automatically update their data by retrieving it from Stata.

How SWordy works

SWordy allows the communication between Word and Stata. This magic is made possible by SWire, a Stata Java plugin. SWire is essentially a server which execute several operation which are requested by external client applications. In this communication architecture, SWordy runs inside Word and SWire runs inside Stata, thus establishing a bridge between Word and Stata.

SWordy and SWire communicates with Stata through an HTTPS connection. In order to allows the communication between Word and Stata, SWire must run in HTTPS mode. You can start the SWire HTTPS in Stata by typing "swire start, https".

Requirements

In order to use SWordy with Microsoft Word you need:

  • Microsoft Word from Office >= 2016
  • Stata >= 13
  • SWire >= 0.2

SWire must be started in HTTPS mode.

SWire

What is SWire?

SWire is a software interface enabling us to query Stata for the executing of basic operations like reading or writing data. The core of SWire is a Stata Java plugin which functions as a server and it can be queried by using the HTTP or, from version 0.2, the HTTPS protocol. SWire is in essence a wrapper around the Stata-Java API and all the functions relating to this API can, therefore, be externally called by clients.

SWordy uses SWire for allowing the communication between Word and Stata.

Minimum required version

SWordy can communicate with SWire only by using the HTTPS protocol. SWire can use HTTPS only from version 0.2, thus you need at least this version number.

How to install

First of all, be aware that SWire is a Stata Java plugin and it can be installed in Stata with version number at least 13. SWire can be downloaded from here. You have to download the binary zip file and uncompress it. In the resulting folder you will find a README file with all the instructions to install it. If you want to be able to use SWire in HTTPS mode in you computer system, you must trust the SWire server certificate. To learn about HTTPS read this, and read this for learning what a TLS/SSL certificate is.

How to run SWire for SWordy

Type this in Stata: swire start, https

Installation

There a two way for installing SWordy into Word:

  • from the Office Store (https://store.office.com)
  • locally

Installation from the Office Store is very easy, but you will need an internet connection when you want to use SWordy because in this case SWordy will be loaded from internet. Instead, local installation is far from easy, but it allows you to use SWordy even if you do not have any available internet connection. Local installation is recommended only for advanced users.

Installation from the Office Store

If you install SWordy from the Office Store, you do not need any of the files you eventually downloaded from any of the official repositories. You only need to follow these steps:

  1. Open Microsoft Word (it must be at least version 2016).
  2. From the "Insert" tab in the ribbon, click on the "Office Add-ins" button.
  3. Select "Store"
  4. Search "SWordy" by Giovanni Lo Magno and install it.

At the end of these steps the SWordy add-in will be installed in Word. You should notice a new "SWordy" tab on the ribbon.

Local installation

Local installation allows you to use SWordy off-line, but is recommended for advanced users only. Ordinary users may prefer instead the installation from the Office Store.

Users that want to install SWordy locally should have a basic understanding of the following topics:

  • HTTPS protocol
  • public key certificate
  • Office add-ins

To install SWordy locally you need to download it from any of the following repositories:

  • SourceForge: https://sourceforge.net/projects/swordy
  • GitHub: https://github.com/lomagno/swordy

If you choose to download SWordy from SourceForge you will obtain a zip file. In this case you have to unzip the downloaded file.

If you want to use GitHub, you can obtain the entire GIT repository, with the advantage to have access to the latest development version or any previous release. You can clone the SWordy repository by typing:

git clone https://github.com/lomagno/swordy

Before starting the local installation, be aware that SWordy is an Office add-in and as all the Office add-ins is composed by a manifest file and a web application. The manifest file describes the add-in by providing Word with all the required informations for the add-in activation. The web application is the core of the add-in and can be hosted in a remote web server or locally.

The SWordy manifest for local installation is the "swordy-manifest-local.xml" file, which is in the "manifest" folder. This file must be loaded by Word, but for security reasons Word can load it only if the manifest is served in an appropriate way. You will learn how to allow Word to load the manifest later.

The web application can be hosted on the internet or on the local system, but in both cases it must be served through the HTTPS protocol. For local installation you can use the SWordy HTTPS server. Before you start this server, you need to install all the Node.js package dependencies (after installling npm, you can type "npm install"). Furthermore, the SWordy HTTPS server needs several private keys and certificates. You can generate them by executing the "generate_certificates.sh" script, which requires OpenSSL. Linux and Mac users can run this script from their shell, but if you are a Windows user you need a shell from which you can run a bash script and openssl commands. In this case you could install Git BASH for Windows after downloading it from https://git-scm.com/download/win. Finally, keep in mind that the public certificate which is used by the SWordy HTTPS server must be trusted by your system. You have to read your operating system documentation to learn how to trust this certificate.

To install SWordy locally, follow these steps:

  1. Install Node.js (https://nodejs.org) and npm (https://www.npmjs.com/).
  2. From your terminal, enter the "swordy" folder (by typing, for example, "cd /mypath/swordy").
  3. Type "npm install" to install all package dependencies (for this step you need an internet connection).
  4. Execute the "generate_certificates.sh" script by typing "./generate_certificates.sh" (in Windows you can do this from Git BASH). This script will generate the following files: swordy_ca.key (the private RSA key for the SWordy certification authority), swordy_ca.crt (the self-signed certificate for the SWordy certification authority), server.key (the private RSA key for the server), server.csr (the certificate signing request for the server) and server.crt (the certificate for the server). Optionally, you can delete the server.csr file because it is used only as an intermediate file in the certification generation process.
  5. Install the certificate swordy_ca.crt in your system. How to install this certificate dependes on your operating system. You can find instructions in the "Installing the certificate" section in this README. The certificate have a validity of about 5 years (1826 days). After this period you have to disinstall the old certificate, run again the generate_certificates.sh script and reinstall the certificate swordy_ca.crt in your system.
  6. Start the HTTPS web server by typing "npm start".
  7. You can test the connection to the SWordy HTTPS server by trying to open https://localhost:3000/ from your web browser. The web browser you should use for this test depends on your operating system. If you use Word for Windows you should execute the test with Internet Explorer, because it is the browser which is internally used by Word.
  8. Now you need to let Word load the "swordy-manifest-local.xml" manifest file, which is contained in the "manifest" folder. The procedure is different according to what Word you use. In the following steps you will find particular instructions.
  9. If you use Word for Windows, you can serve the manifest file from a shared folder. To do this, you have to set the "manifest" folder as shared. To this end follow these steps:
    • right-click on the "manifest" folder which is under the "swordy" main folder;
    • choose "Properties";
    • click on the "Sharing" tab;
    • open the "Advanced sharing" dialog window by clicking on the "Advanced sharing..." button;
    • check the checkbox "Share this folder";
    • type a name for the shared folder in the "Share name" text field: for example type "SWordy manifest folder";
    • click on the the "Permissions" button to open a dialog window for setting the permissions and here allow all the permissions (including read and write permissions);
    • close Word and relaunch it;
    • in Word, from the "File" tab, choose "Options";
    • select "Trust Center";
    • click on the "Trust Center Settings...." button;
    • select "Trusted Add-in Catalogs";
    • type the network path to the shared folder (for example: "\\MYUSERNAME-PC\SWordy manifest folder") in the "Catalog Url" text field and then click on the "Add catalog" button to add this path to the list of the trusted catalog addresses (if you do not know the network path to the shared folder, then right-click on that folder, choose "Properties", select the "Sharing" tab and read the network path from there);
    • in the list of the trusted catalog addresses, find the network path you just added and check the "Show in Menu" checkbox for it;
    • close all the Word dialog windows you opened for setting the options by clicking on the "OK" buttons and accept every change;
    • close Word and restart it;
    • select the "Insert" tab on the ribbon, click on the "Add-ins" button and choose "My Add-ins": this will open the dialog window you will use to add the SWordy add-in;
    • click on "Shared folder";
    • select "Swordy (local)" and click on the "Add" button (if you do not see the SWordy logo for the "SWordy (local)" add-in, probably the SWord HTTPS server is not started and, before adding the SWordy add-in, you should start this server by typing "npm start" from the terminal).

At the end of these steps the SWordy add-in will be installed in Word. You should notice a new "SWordy" tab on the ribbon.

Insert Stata data

Two types of Stata can be inserted in the Word document:

  • numeric scalar
  • matrix

Data is inserted at the cursor position. If a documentation selection is active, that selection will be replaced by the inserted data.

Insert scalar

Stata numerical scalars can be inserted by using the "Insert scalar" taskpane. This taskpane can be opened by clicking on the "Insert scalar" button on the ribbon, under the "SWordy" tab.

In order to insert a scalar in the Word document you must:

  • insert the scalar name: a name for an existing numeric scalar is required;
  • choice the number of decimals: this is to set how many decimals will be used to format the inserted number.

Once these input data are correctly set, you can insert the desired scalar by clicking on the "Insert scalar" button on the taskpane. This button is disabled until all the required input data are set.

The scalar will be inserted at the current position and will replace any current selection.

See also

  • Insert Stata data
  • Insert matrix

Insert matrix

Stata matrices can be inserted by using the "Insert matrix" taskpane. This taskpane can be opened by clicking on the "Insert matrix" button on the ribbon, under the "SWordy" tab.

In order to insert a matrix in the Word document you must:

  • insert the matrix name: a name for an existing matrix is required
  • choice the number of decimals: this is to set how many decimals will be used to format the inserted numbers.

Once these input data are correctly set, you can insert the desired matrix by clicking on the "Insert matrix" button on the taskpane. This button is disabled until all the required input data are set.

The matrix will be inserted at the current position as a Word table and will replace any current selection.

Bindings

What is a binding?

A binding is a placeholder in the Word document which is created for being automatically replaced by a Stata data. You can use bindings for creating an automatic document, namely a document with numeric values and tabular data that can be automatically retrieved from Stata. An automatic report relieves the user from the burden of manually update reports that share a common format but are different only for the numeric values they convey.

The "Bindings" taskpane

Bindings are created and managed through the "Bindings" taskpane. You can open this taskpane by clicking on the "Bindings" button in the ribbon, under the "SWordy" tab. This taskpane has two sections:

  • Create: it allows the user to create new bindings to scalar or matrix Stata data;
  • Manage: it allows to list, search and delete the bindings, besides updating their content.

Creating new bindings

New bindings can be created from the "Create" section of the "Bindings" taskpane. There are two types of data you can bind to: scalar and matrix.

If you want to create a binding for a scalar you have to select a piece of text first (no matter the content of the text); then from the "Create" section select "scalar" as binding type and set the scalar name and the number of decimals you want to be used for formatting the numeric output format; finally click on the "Bind" button.

If you want to create a binding for a matrix you have to select a Word table; then from the "Create" section select "matrix" as binding type and set the scalar name, the row and the column numbers where you want to start inserting the matrix and the decimals for the desired numeric output format; finally click on the "Bind" button.

Managing existing bindings

Existing bindings can be managed from the "Manage" section of the "Bindings" taskpane. You can switch to this section by clicking on the "Manage" button on the taskpane. By using this section you can list, search and sort the bindings that exist in the document. Furthermore you can sync Stata data with the content of your bindings.

A toolbar provides a set of buttons for accessing the managing functions. The following is a list of these buttons, together with a short explanation of their associated functions:

filter the bindings by their name;
check or deselect all the bindings;
sort bindings;
refresh the bindings list
delete the selected bindings
sync data for the selected bindings

Sync document

You can sync all the bindings of your Word document by clicking on the "Sync document" button on the ribbon, under the "Swordy" tab. After this button has been clicked, the "Sync document" taskpane will be opened. This taskpane will inform you about the execution status of the sync and provide you with a button for repeating the sync.

If you want to sync only a subset of the existing bindings, you can do this from the Bindings taskpane.

Check Stata connection

SWordy communicates with Stata through SWire for retrieving data. You can check if the connection with Stata is functioning by clicking on the "Check Stata connection" button on the ribbon, under the "Swordy" tab. A taskpane will open with a "Check" button to repeat the check. A report on the taskpane will inform you if the connection is ok and, if not, it will provide you with instructions to solve the problem.

Troubleshooting

Uninstalling

From Word for Windows or Mac

If you want to uninstall SWordy which you installed on Word for Windows or Mac, follow this instructions:

  • in Word, click on the "Insert" tab on the ribbon
  • click on the "Add ins" button and select "My add-ins" from the dropdown menu item
  • click on "My add-ins"
  • right-click on the SWordy add-in and select "Remove".

From Word Online

When you load an add-in in Office Online, its manifest (a manifest is a file which describes the add-in) is stored in the browser's local storage. Thus you have to clear the browser's cache in order to uninstall SWordy from Office Online. To clear the cache, please refer to you browser's documentation.

Notice that all the other add-ins you previously installed will be removed from your browser.
